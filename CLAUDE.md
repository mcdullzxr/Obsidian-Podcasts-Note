# Obsidian Podcast Plugin

## 产品定位

把播客从"听过就忘"变成"可搜索、可关联、可复习"的知识资产。

核心价值：音频播客 → AI 转录 → 结构化知识笔记。

## 已确认的决策

### 目标平台（按优先级）

1. **小宇宙**（中文播客主阵地）
2. **RSS 泛用播客**（Pocket Casts、Overcast 等）
3. **Spotify**
4. YouTube 播客不是重点（已有成熟方案）

### 交互方式

- 双入口：浏览器剪藏 + Obsidian 内粘贴链接
- MVP 先做「粘贴链接」模式

### AI 策略

- **AI 语音转录是核心功能**，不是兜底方案（目标平台普遍没有现成文字稿）
- **BYOK 模式**（Bring Your Own Key）：用户自带 API Key，插件不提供 AI 服务
- 首版支持 Whisper（转录）+ LLM（摘要/提炼）

### 商业模式：BYOK（用户自带 API Key）

**为什么选 BYOK：**
- Obsidian 生态惯例（Smart Connections、Text Generator、Copilot for Obsidian 等知名 AI 插件全部采用）
- 用户重视隐私，不希望音频数据经过第三方服务器
- 开发者 0 运营成本，专注功能本身
- 避免后端开发、账号系统、支付系统、合规等复杂度

**用户成本参考（60 分钟中文播客）：**
- OpenAI 方案：约 ¥2.7/期（Whisper ¥2.6 + GPT-4o-mini ¥0.1）
- DeepSeek + 硅基流动方案：约 ¥1/期以内

**降低用户门槛的措施：**
1. **兼容多 AI 服务商**：OpenAI / DeepSeek / 硅基流动 / Groq / 自定义 OpenAI 兼容 endpoint
2. **清晰的配置指南**：README 中手把手教用户注册和配置 API Key
3. **成本预估**：每次生成前显示预计消耗
4. **本地缓存转录结果**：避免重复付费转录（可省 80% 费用）

**未来商业化路径（非首版目标）：**
- 免费版保留所有核心功能 BYOK
- Pro 订阅增值功能：浏览器扩展、仪表盘、复习系统、模板市场、云同步配置
- 不做 SaaS（转录代付费模式）

### 脑图格式

- Markdown 大纲为默认输出
- Canvas 脑图为可选导出

### 笔记设计原则

- 纯 Markdown + YAML Frontmatter（卸载插件后笔记仍可用）
- 不需要分级处理：每次使用都做全量处理
- 模板可自定义，保存路径可配置

### 目标

- 先自用，成熟后发布到 Obsidian 社区插件市场

## 核心功能（按优先级）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 播客链接解析 | 输入小宇宙/Spotify/RSS 链接 → 自动提取元数据 |
| P0 | AI 逐字稿生成 | Whisper 转录，带时间戳 + 说话人区分 |
| P0 | AI 智能提炼 | 摘要 + 知识点卡片 + 案例提取 + 自动标签 |
| P1 | 结构化大纲 | 按话题分段的层级大纲 |
| P1 | Canvas 脑图导出 | 可选导出为 Obsidian Canvas |
| P1 | 知识原子化 | 知识点 → 独立笔记 + `[[双链]]` 回溯 |
| P2 | 浏览器扩展 | 在小宇宙/Spotify 网页版一键剪藏 |
| P2 | 播客库仪表盘 | Dataview 查询所有播客笔记 |

## 笔记输出结构

### 文件路径

`Podcasts/YYYY-MM-DD-标题.md`（扁平结构，路径可在设置中自定义）

### 模板

```markdown
---
title: "张三谈第一性原理思维"
podcast: "随机波动"
host: "主持人名"
guests: ["张三"]
date: 2026-04-10
source: "https://www.xiaoyuzhoufm.com/episode/xxx"
duration: "01:15:30"
tags: [播客, 第一性原理, 思维模型]
created: 2026-04-16
---

## 📋 摘要

张三在本期节目中分享了第一性原理思维在创业中的应用。
他通过 SpaceX 和 Tesla 的案例，说明了如何从基本事实出发
重新推导解决方案，而非依赖类比推理。

## 💡 知识点

> [!abstract] 第一性原理（First Principles Thinking）
> 将复杂问题拆解到最基本的事实和假设，然后从这些基础出发重新推导解决方案。源自亚里士多德，被 Elon Musk 广泛推广。
> ⏱️ 12:34

> [!abstract] 类比推理的陷阱
> 类比推理是"别人怎么做我就怎么做"，省力但可能继承错误假设。
> ⏱️ 23:10

## 📖 案例

> [!example] SpaceX 火箭成本
> 传统火箭成本 6500 万美元，Musk 从原材料出发重新计算，发现材料成本仅占 2%，据此重新设计了供应链。
> ⏱️ 30:00

## 🗺️ 内容大纲

- 开场：为什么要谈第一性原理 (00:00)
  - 背景：创业者面临的思维惯性
- 什么是第一性原理 (12:34)
  - 定义与起源
  - 与类比推理的区别
- 实际应用案例 (28:00)
  - SpaceX 火箭成本
  - Tesla 电池供应链
- 如何训练这种思维 (52:00)
  - 五个步骤
  - 常见误区

## 📝 逐字稿

**[00:00] 主持人：** 大家好，欢迎来到随机波动...
**[00:15] 张三：** 谢谢邀请。今天我想聊聊...
**[02:30] 主持人：** 你最初是怎么接触到这个概念的？
...
```

### 设计理念

- **单文件完整收录**：摘要、知识点、案例、大纲、逐字稿全部在同一篇笔记中
- **从上到下信息密度递减**：摘要 → 知识点 → 案例 → 大纲 → 逐字稿
- **知识点用 Callout 呈现**：`> [!abstract]` 简洁版，只含定义 + 时间戳
- **案例独立成块**：`> [!example]` 方便快速浏览
- **逐字稿区分说话人**：播客多人对话特性，`**[时间戳] 说话人：**` 格式

### 标签策略

- AI 优先匹配用户 Vault 中已有的标签
- 没有合适已有标签时，AI 自主新建
- 固定包含 `播客` 标签

## 竞品参考

| 产品 | 做了什么 | 差异化空间 |
|------|----------|-----------|
| Snipd | 听播客时 AI 高亮片段，导出到笔记 | 独立 App，不在 Obsidian 生态内 |

**我们的差异化：原生 Obsidian 插件 + 播客专用 + 知识管理导向**

## 开发路线

- **Phase 1（MVP）**：Obsidian 插件 → 粘贴小宇宙/RSS 链接 → AI 转录 → 知识笔记
- **Phase 2**：支持 Spotify 链接解析
- **Phase 3**：Canvas 脑图导出 + 知识原子化（独立笔记+双链）
- **Phase 4**：浏览器扩展（一键剪藏）
- **Phase 5**：播客库仪表盘 + 复习系统
